散逸している審議会にアクセスしやすくするためのポータルサイト

データの粒度はいくつかある
- 開催状況(開催日と公式ページにアクセスするための出店等のリンク)
- 資料議事録等も網羅したもの
- 出席者とか名簿とかも網羅したもの

など

基本的にプログラムで更新確認やパースなどを行ってデータを作成する
一部、人力に頼る部分も存在する。
それように管理画面も作成したりしてデータの編集をやりやすくしたり、直接jsonをいじって編集したりとかする

データのモデル
- 会議体そのもの
  - 名称
  - 親組織
  - 公式ページへのリンク
  - 開催状況の確認に使うためのリンク
- 名簿(会議体に対し複数存在)
  - 更新とかがあるので何日時点の情報ですっていうような書き方をする
  - 項目
    - 何日時点の情報か
    - 以下のオブジェクトの配列
      - id(人物登録なしの場合は単に名前の文字列)
      - 役職
        - 会議体の中での話
        - 社長とかではなく委員とか委員長とか
- 開催記録
  - 基本的な情報
    - この項目がまず第一段階
      - 開催日時
      - それにアクセスするためのリンク
  - 出席者
  - 逐語録
  - 議事要旨
  - 資料たち
- 答申や報告書などの文書群
  - pdfへのリンクでいいかな
